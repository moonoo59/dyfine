## 📐 2. DB 스키마 변경 계획

### 2-1. `loans` 테이블 확장 (S1 대출현황 시트 대응)

```
현재 loans 테이블 컬럼:
  id, household_id, name, principal_original, start_date, maturity_date,
  term_months, repayment_type, interest_pay_day, day_count_convention,
  rounding_rule, linked_payment_account_id, is_active, created_at

추가할 컬럼:
  + bank_name TEXT           -- 대출 취급 은행명 (신한은행, NH농협 등)
  + repayment_priority INT   -- 사용자 정의 상환 순위 (1이 최우선)
```

**마이그레이션**: `0021_loan_extend.sql`

### 2-2. `securities` 테이블 확장 (S4 테마별 PF 대응)

```
현재 securities 테이블 컬럼:

추가할 컬럼:
  + theme TEXT DEFAULT 'index'   -- 테마 분류 (index, dividend, bond, gold, china, theme, individual, cash)
```

### 2-3. `investment_targets` 테이블 신규 (S4 목표PF 시트 대응)

```sql
CREATE TABLE investment_targets (
    id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    household_id UUID REFERENCES households(id) ON DELETE CASCADE,
    theme TEXT NOT NULL,           -- 'index', 'dividend', 'bond', 'gold', 'china', 'theme', 'individual', 'cash'
    target_weight NUMERIC(5,2),    -- 목표 비중 (%) 예: 30.00
    label TEXT,                    -- 한글 라벨 (예: '지수+채권')
    currency_type TEXT DEFAULT 'all', -- 'krw', 'usd', 'all'
    sort_order INT DEFAULT 0,
    UNIQUE(household_id, theme, currency_type)
);
```

**설명**: 사용자가 "지수+채권 30%, 배당 35%, 금 15%, 중국 20%" 이런 식으로 목표 비중을 직접 설정합니다.

### 2-6. `pet_care_logs` 테이블 신규 (S3 무무누리 훈트가르텐 대응)

> 훈트가르텐 이용 내역은 일반 거래(transaction)가 아닌 **"특정 소비를 기록한다"** 는 개념으로,  
> 입퇴실 일시·이용일수·초과시간·요금 항목 등을 독립적으로 추적하는 전용 로그 테이블입니다.

```sql
CREATE TABLE pet_care_logs (
    id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    household_id UUID REFERENCES households(id) ON DELETE CASCADE,
    visit_number INT,                  -- 이용 회차 (1, 2, 3...)
    check_in_date DATE NOT NULL,       -- 입실일
    check_in_time TIME,                -- 입실시간
    check_out_date DATE,               -- 퇴실일
    check_out_time TIME,               -- 퇴실시간
    stay_days INT,                     -- 이용일 수
    extra_hours_before9 NUMERIC(4,1) DEFAULT 0,  -- 초과시간(9시 이전 입실)
    extra_hours_after18 NUMERIC(4,1) DEFAULT 0,  -- 초과시간(18시 이후 퇴실)
    base_fee NUMERIC(18,2) DEFAULT 0,  -- 기본요금
    nuri_fee NUMERIC(18,2) DEFAULT 0,  -- 누리 기본요금 (20kg이상)
    extra_fee_morning NUMERIC(18,2) DEFAULT 0,  -- 초과요금(09시 이전)
    extra_fee_evening NUMERIC(18,2) DEFAULT 0,  -- 초과요금(18시 이후)
    total_fee NUMERIC(18,2) DEFAULT 0, -- 총 요금 (합산)
    memo TEXT,                         -- 비고
    linked_entry_id BIGINT REFERENCES transaction_entries(id) ON DELETE SET NULL,  -- 정산 시 연결된 거래 전표
    created_at TIMESTAMPTZ DEFAULT now()
);
```

**설명**: 엑셀의 훈트가르텐 이용 내역 시트를 그대로 재현합니다.  
- 이용 기록을 남길 때는 `pet_care_logs`에 INSERT만 합니다 (돈이 나가는 건 아님).  
- 실제 결제/정산이 발생하면 일반 거래(transaction_entries)를 만들고, `linked_entry_id`로 연결합니다.  
- 향후 차량 정비 로그, 병원 진료 로그 등 비슷한 "특정 소비 기록"이 필요하면 같은 패턴으로 확장 가능합니다.

**⚙️ 자동 계산 수식 (프론트엔드 `useMemo`로 구현)**:  
사용자는 **입실일시, 퇴실일시, 기본요금 단가**만 입력하면, 아래 필드는 엑셀 수식과 동일하게 **자동 계산**됩니다.

```
■ 이용일 수 (stay_days)
  = (퇴실일 - 입실일) 일수 차이. 당일 퇴실이면 1일.

■ 초과시간 — 9시 이전 입실 (extra_hours_before9)
  = IF(입실시간 < 09:00, 9시 - 입실시간, 0)  (시간 단위)

■ 초과시간 — 18시 이후 퇴실 (extra_hours_after18)  
  = IF(퇴실시간 > 18:00, 퇴실시간 - 18시, 0)  (시간 단위)

■ 총 초과시간 (total_extra_hours)
  = extra_hours_before9 + extra_hours_after18

■ 무무 기본요금 (base_fee)  — 사전 설정된 단가 테이블 참조
  = 20kg 미만 1박 기준 55,000원 × stay_days

■ 누리 기본요금 (nuri_fee)
  = 20kg 이상 1박 기준 66,000원 × stay_days

■ 초과요금 — 09시 이전 입실 (extra_fee_morning)
  = 11,000원 × extra_hours_before9 (시간당)

■ 초과요금 — 18시 이후 퇴실 (extra_fee_evening)
  = 6,600원 × extra_hours_after18 (시간당)

■ 총 요금 (total_fee)
  = base_fee + nuri_fee + extra_fee_morning + extra_fee_evening
```

> **단가(55,000 / 66,000 / 11,000 / 6,600)는 설정(Settings)에서 수정 가능하도록** 별도 상수 또는 config로 관리하여, 요금이 변경되어도 코드를 건드리지 않습니다.
