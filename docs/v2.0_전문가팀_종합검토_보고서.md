## ⚡ [Backend] 백엔드 개발자 — RPC 정합성 검토

### [Critical] `record_trade` RPC 스키마 충돌 (0014)

```sql
-- 0014_investment_core.sql 문제점:

-- 1) L102: transaction_entries에 'amount', 'is_confirmed' 컬럼 사용
--    → 실제 스키마(0002)에는 이 컬럼이 없음!
INSERT INTO transaction_entries (
    household_id, occurred_at, entry_type, amount, memo, is_confirmed
) VALUES (...);

-- 2) L113: transaction_lines에 'is_increase', 'category_id' 컬럼 사용
--    → 실제 스키마(0002)에는 이 컬럼이 없음!
INSERT INTO transaction_lines (entry_id, account_id, amount, is_increase) ...
INSERT INTO transaction_lines (entry_id, category_id, amount, is_increase) ...

-- 3) L129: accounts에 'balance' 컬럼 사용
--    → 실제 스키마(0001)에는 'opening_balance'만 존재!
UPDATE accounts SET balance = balance + ... WHERE id = p_account_id;
```

**[해결 방안]**: `record_trade` RPC를 `create_transaction` RPC(0004)와 동일한 패턴으로 재작성:
- `transaction_entries` INSERT 시 `amount`, `is_confirmed` 제거
- `transaction_lines` INSERT 시 `is_increase` 제거, `category_id` 대신 entry의 `category_id` 사용
- 계좌 잔액 직접 갱신 로직 제거 (`v_account_balance_actual` View 활용)

| `confirm_auto_transfer` | 0009 | ✅ 정상 | 전표+라인+인스턴스 확인 원자 처리, **합계=0 보장** |
